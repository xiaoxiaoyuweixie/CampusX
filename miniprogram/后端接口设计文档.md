# 校易通微信小程序后端接口设计文档

> 说明：本文档基于当前前端小程序页面结构、Mock 数据和交互逻辑整理，目标是为“校园二手/资源交易”类小程序设计一套可落地的后端接口。
>
> 当前前端涉及的核心能力包括：登录/用户信息、首页推荐、分类浏览、商品详情、发布商品、收藏、消息会话、聊天、个人中心、我的发布、设置等。

---

## 1. 项目角色与核心业务对象

### 1.1 业务角色
- 普通用户：浏览商品、收藏、聊天、发布商品
- 卖家：发布商品、管理自己的商品、查看聊天消息
- 管理员（可选）：审核内容、管理举报、处理违规信息

### 1.2 核心业务对象
- 用户 `User`
- 商品/发布信息 `Product`
- 商品分类 `Category`
- 收藏 `Favorite`
- 会话 `ChatSession`
- 聊天消息 `ChatMessage`
- 系统通知 `SystemNotice`
- 举报/反馈 `Report`（建议预留）
- 设置 `Settings`

## 2. 接口设计原则

1. 统一前缀：`/api/v1`
2. 统一返回格式：成功/失败都返回结构化 JSON
3. 分页统一：`page`、`pageSize`、`total`
4. 鉴权统一：登录后通过 `token` 访问受保护接口
5. 状态码统一：HTTP 状态码 + 业务码双层表达

### 2.1 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "requestId": "b7f3c8c1"
}
```

### 2.2 通用错误码建议
- `0`：成功
- `40001`：参数错误
- `40004`：未登录或 token 无效
- `40003`：无权限
- `40400`：资源不存在
- `40900`：数据冲突
- `50000`：服务器内部错误

## 4. 认证与用户接口

### 4.1 微信登录
**接口**：`POST /api/v1/auth/wechat/login`

用于前端调用 `wx.login()` 后，将 `code` 发送到后端，后端换取 `openid/session_key` 并签发 token。

#### 请求参数
```json
{
  "code": "wx.login 返回的临时 code",
  "encryptedData": "可选",
  "iv": "可选"
}
```

#### 响应示例
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "jwt-token",
    "expiresIn": 7200,
    "user": {
      "id": "u1001",
      "nickname": "校园用户",
      "avatar": "",
      "verified": false,
      "school": "西南大学"
    }
  }
}
```

#### 业务说明
- 首次登录自动创建用户
- 后续登录返回已有用户信息
- `token` 建议使用 JWT 或自定义 token

---

### 4.2 获取当前用户信息
**接口**：`GET /api/v1/users/me`

#### 请求头
`Authorization: Bearer <token>`

#### 响应示例
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "u1001",
    "nickname": "校园用户",
    "avatar": "https://...",
    "verified": true,
    "school": "西南大学",
    "phone": "138****8888",
    "gender": "unknown",
    "bio": "",
    "createdAt": "2026-06-20T08:00:00Z"
  }
}
```

---

### 4.3 更新当前用户信息
**接口**：`PUT /api/v1/users/me`

#### 请求参数
```json
{
  "nickname": "新的昵称",
  "avatar": "头像地址",
  "bio": "个人简介",
  "phone": "手机号",
  "school": "学校名称"

```

#### 返回

---

### 4.4 退出登录
**接口**：`POST /api/v1/auth/logout`

#### 作用

- 清理登录态

## 6. 商品/发布接口

### 6.1 获取商品列表
**接口**：`GET /api/v1/products`

#### 常用查询参数
- `keyword`：关键词搜索
- `category`：分类 id
- `campus`：校区
- `sort`：`composite` / `newest` / `price`
- `page`：页码，默认 1
- `pageSize`：每页数量，默认 10
- `excludeMine`：是否排除自己发布的商品

#### 示例
`GET /api/v1/products?keyword=iPad&sort=newest&page=1&pageSize=10`

#### 响应示例
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "title": "九成新 iPad Air 4 64G 银色",
        "price": 2399,
        "category": "digital",
        "campus": "荣昌校区",
        "location": "荣昌校区·玉兰园三舍门口",
        "views": 326,
        "cover": "https://...",
        "sellerId": "u001",
        "sellerName": "小桂",
        "publishedAt": "2小时前"
      }
    ],
    "page": 1,
    "pageSize": 10,
    "total": 1
  }
}
```

#### 说明
- 首页、分类页、我的发布页都可复用该接口
- 搜索建议走服务端，避免前端只做本地过滤

## 15. 前后端对接建议

### 15.1 首页
当前前端使用本地 Mock 数据，后端接入后建议改为：
- 启动时请求分类列表
- 请求推荐商品列表
- 搜索时走后端搜索接口

### 15.2 分类页
- `activeId` 改为分类接口返回的 `id`
- 排序逻辑建议后端处理，避免前端排序与分页冲突

### 15.3 发布页
- 先上传图片，再提交商品
- 发布成功后刷新“我的发布”和首页

### 15.4 详情页
- 商品详情接口需返回 `isFavorited`
- 收藏按钮直接调用收藏接口
- 联系卖家时先创建/获取会话，再跳转聊天页

### 15.5 消息页
- 会话列表从后端拉取
- 系统通知单独接口返回
- 未读数用于右上角红点或角标

## 17. 备注

当前前端仍大量依赖本地 Mock 数据和本地缓存，因此后端正式接入时，建议优先改造以下数据流：
1. 商品数据由后端统一提供
2. 收藏、我的发布、用户信息改为账号维度存储
3. 聊天内容由服务端持久化
4. 图片统一走对象存储

这样可以保证多端同步、数据持久化和后续扩展能力。
